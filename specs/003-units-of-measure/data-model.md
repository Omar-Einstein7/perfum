# Data Model: Units of Measure

## Entity: Unit

| Field | Type | Required | Unique | Notes |
|-------|------|----------|--------|-------|
| `id` | String (MongoDB ObjectId) | ✅ | ✅ | Auto-generated by backend |
| `name` | String (max 100 chars) | ✅ | ✅ | Display name, e.g. "Kilogram" |
| `abbreviation` | String (max 10 chars) | ✅ | ✅ | Short symbol, e.g. "kg" |
| `type` | Enum | ✅ | No | One of: weight, volume, count, length, area, time, other |
| `description` | String (max 500 chars) | No | No | Free-text description |
| `active` | Boolean | ✅ | No | Soft-delete flag; `true` by default |
| `createdAt` | DateTime | ✅ | No | Set by backend |
| `updatedAt` | DateTime | ✅ | No | Updated by backend |

## Relationships

- **Unit → Material**: A Unit can be referenced by zero or more Materials (via `material.unitId`).
- When a Unit is referenced by any Material, deactivation is blocked (backend returns 409).

## API Contract

### GET /units?page=1&limit=20&search=kg&active=true

**Response:**
```json
{
  "data": [
    {
      "_id": "64a1b2c3d4e5f6a7b8c9d0e1",
      "name": "Kilogram",
      "abbreviation": "kg",
      "type": "weight",
      "description": null,
      "active": true,
      "createdAt": "2024-01-01T00:00:00.000Z",
      "updatedAt": "2024-01-01T00:00:00.000Z"
    }
  ],
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 5,
    "pages": 1
  }
}
```

### GET /units/:id

**Response:** Single unit object (same shape as above), or 404.

### POST /units

**Request:**
```json
{
  "name": "Liter",
  "abbreviation": "L",
  "type": "volume",
  "description": "Metric unit of volume"
}
```

**Response:** Created unit with `_id`, 201 status. 409 on duplicate name/abbreviation. 422 on validation failure.

### PUT /units/:id

**Request:** Same shape as POST (partial update supported).
**Response:** Updated unit, 200 status. 409 on duplicate. 404 if not found.

### DELETE /units/:id

**Response:** 200 with `{ "success": true }` on soft-delete. 409 if referenced by materials. 404 if not found.

## Validation Rules

| Field | Rule |
|-------|------|
| name | Required, 1–100 chars, trimmed, unique (case-insensitive) |
| abbreviation | Required, 1–10 chars, trimmed, uppercase by convention, unique |
| type | Required, must be one of the 7 enum values |
| description | Optional, 0–500 chars |
| active | Default `true`; only `false` via soft-delete endpoint |
